Ejercicios 05 JavaScript

¿Que es Javascript?

Es un lenguaje de programación ligero, interpretado, o compilado justo-a-tiempo. Si bien es más conocido como lenguaje de script principalmente usado para páginas web, es usado en variados entornos fuera del navegador. Es un lenguaje basado en prototipo, multi-paradigma, dinámico, de un solo hilo y orientada a objetos.

¿Qué hace JavaScript?

JavaScript es principalmente usado para crear interactividad dinámica en sitios web. Se pueden crear cosas como galerías de imágenes, diseños fluctuantes y respuestas a pulsaciones de botones, también se puede utilizar para crear aplicaciones basadas en base de datos, animaciones 2D, gráficos 3D entre muchas otras cosas.

¿Qué es el DOM?

Es el modelo de objeto e documento, la cual es una interfaz de programación para los documentos HTML y XML. Facilita una representación estructurada del documento y define la manera en la que los programas puedan acceder, al fin de modificar, tanto estructura, como contenido y estilo. El DOM representa el documento como un grupo de nodos y objetos estructurados que tienen métodos y propiedades.

¿Qué es un APIs Web?

Las APIs o las Interfaces de Programación de Aplicación (Aplication Programming Interface) son construcciones de lenguajes de programación que permiten crear funcionalidades complejas de manera que sea más sencillo. Abstraen el código complejo en una sintaxis fácil de usar.

¿Cómo se declara una variable y cuáles son sus principales características?

Con la palabra reservada ‘var’, la forma de definir una variable es primero escribir la palabra var, seguido de un nombre para la variable, después con el signo = se le asigna un valor a la variable, puede ser un valor numérico, flotante, booleano o de texto, ente otros. Var numero = 10

¿Cómo se declara una condición y cuáles son sus principales características?

Las condiciones se declaran con las palabras ‘if, else’ siendo if (condición){ bloque de código } else {Bloque de código} mientras se cumpla la condición del if, se ejecuta el bloque de código, si no se cumple la condición se ejecuta el bloque de código del else.

¿Cómo se declara bucle y cuáles son sus principales características?

El bucle while, es un bucle que se repetirá mientras su condición sea verdadera o falsa, su sintaxis seria: while (condición) {}

El bucle for de manera similar al while se repite mientras su condición se cumpla: for(inicio, condición,incremento)

¿Cómo se declara una función y cuáles son sus principales características?

La función se declara con la palabra reservada function(parámetros){bloque de código}

La función cada vez que es llamada ejecuta un bloque de código, la función puede tener parámetros los cuales son valores que pueden llevar a un resultado o a otro, los parámetros son enviados al momento de llamar la función

¿Cómo se puede leer un evento con JavaScript?

La forma en que se produce un evento es al momento de presionar un botón, o una tecla, también al momento de cargar una pagina.

¿Qué eventos podemos leer con JavaScript menciónales y descríbalos?

  • Onblur: un elemento pierde el foco
  • Onchange: Un elemento ha sido modificado
  • Onclick: Pulsar y soltar botón
  • Ondbclick: Pulsar dos veces seguidas con el raton
  • Onfocus: Un elemento obtiene el foco
  • Onkeydown: pulsar una tecla y no soltarla
  • Onkeypress: pulsar una tecla
  • Onkeyup: Soltar una tecla pulsada
  • Onload: página cargada completamente
  • Onmousedown: Pulsar un botón del ratón y no soltarlo
  • Onmousemove: Mover el raton
  • Onmaouseout: Cuando el ratón sale de un elemento
  • Onmouseover: Cuando el raton entra en un elemento
  • Onmouseup: Soltar el botón del raton
  • Onreset: Inicializar el formulario
  • Onrezise: modificar el tamaño de la ventana
  • Onselect: Seleccionar un texto
  • Onsubmit: Enviar formulario
  • Onunload: Se abandona la pagina

Mencionan un ejemplo donde JavaScript sea ejecutado del lado del cliente

Los lenguajes del lado del cliente son aquellos que se pueden ejecutar desde el navegador sin necesidad de recargar desde el servidor, por lo que el servidor envía código y es tarea del navegador interpretarlo. Ya que se ejecuta desde el navegador se puede creare lo que sea que permita javascript, desde juegos, animaciones, interactividad con los usuarios, etc.

Mencionan un ejemplo donde JavaScript sea ejecutado del lado del servidor

A pesar de que en un principio JavaScript es un lenguaje del lado del cliente, actualmente también es un lenguaje por parte del servidor, los servidores con los cuales puedes llegar a utilizar JavaScript son Node.js,EjScript, RingoJs y AppEgineJs. Un problema que se puede resolver gracias a esto vendría siendo la forma en la que se pueden hacer conexiones con el servidor, en lugar de generar un hilo con la OS para la conexión; Cada ejecución dispara una ejecución de evento.

¿Qué tipo de errores podemos encontrar en JavaScript?

Los errores mas comunes de JavaScript son:

  • Uncaught TypeError: cannot read property: Ocurrre chrome lee una propiedad o llama a un método de un objeto indefinido.
  • typeError: “undefine” is not an object: Ocurre cuando en Safari lees una propiedad o llamas un método de un objeto indefinido.
  • TypeError: nut is not an Object: Ocurre cunado lees una propiedad o llamas un método de un objeto nulo
  • Unknown: Script error: Este error nos avisa cuando se origina un error de un archivo JavaScript servido desde un origen diferente
  • TypeError: Object doesn’t suppot property: Este es un error que ocurre en Internet Explorer cuando haces uso de un método indefinido

¿De qué manera podemos ver los errores de JavaScript en el navegador?

Depende del navegador, es diferente la forma de ver los errores:

  • Mozilla Firefox, presionas las teclas Ctrl + Shift +J en una PC o Command + Shift + J si estas en una Mac.
  • Google Chrome, presionas las teclas Ctrl + Shift +J en una PC o Command + Shift + J si estas en una Mac.
  • Apple Safari, seleccionas el menú Safari -> Preferencias -> Avanzado -> "Mostrar menú de desarrollo en barra de menú".
  • Microsoft Internet Explorer, en el menú principal seleccionas Herramientas -> Opciones de Internet -> Avanzado luego quitas el cheque en la opción que dice "Disable script debugging" y seleccionas la que dice "Diplay a Notification About Every Script Error".
  • Opera, en el menú principal seleccionas Herramientas -> Avanzado -> Consola de Errores.

¿Cuántas versiones de JavaScript han existido menciona sus principales características?

  • 1995: Primeras versiones de JavaScript, todavía con nombres provisionales como Mocha , LiveScript
  • 1997: Definicion del primer estándar JavaScript a cargo de Ecma International que fue denominado ECMA-262 first edition también denominado 1.2
  • 1998: Aparicion del segundo estándar JavaScript denominado ECMA-262 también denominado JavaScript 1.3
  • 2000: Aparición de la especificación del estándar JavaScript denominado ECMA-262 third edition también denominado JavaScript 1.5
  • 2010. Aparición de la especificación del estándar JavaScript denominado ECMA-262 fifth edition también denominado JavaScript 1.8.5
  • 2019. Fecha prevista para ECMA-262 sixth edition

¿Cuántas versiones de ECMAScript han existido menciona sus principales características?

  • 1997 primera edición
  • 1998: Cambios editoriales para mantener la especificación completa alineada con el estándar internacional
  • 1999: Se agregan expresiones regulares, mejor manejo de strings, nuevo control de declaraciones, manejo de excepciones, definición mas estricta de errores, formato para salidas numéricas.
  • 2009 quinta edición: Se agregó modo estricto, librería para el soporte de JSON.
  • 2011 edición 5.1
  • 2015 edición 6: Cambios en sintaxis, se agrego loop for, funciones de dirección, datos binarios, colecciones y proxies
  • 2016 edición 7: agrega operador exponencial (**), incorpora método Array.proptotype.includes()
  • 2017 edición 8: Inclusión de de constructores async/await
  • 2018 edición 9: Inclsion de operadores rest/spread, iteración asincrónica.
  • 2019 edición 10: incorporó Array.flat(), Array.flatMap(), String.trimStart(), String.trimEnd(), errores opcionales en el bloque catch, Object.fromEntries(), Symbol.description

¿Qué es WebGL?

API implementada en JavaScript para la renderizacion de gráficos 3D dentro de cualquier navegador. No necesita de plugins adicionales a cualquier plataforma que soporte OpenGL 2 o OpenGL ES 2.0. Los elementos de WebGl se pueden combinar con otros elementos html y estar compuesto con otras partes de la página o del fondo.

¿Qué es un webSocket?

WebSockets es una tecnología avanzada que hace posible abrir una sesión de comunicación interactiva entre el navegador del usuario y un servidor. Con esta API, puede enviar mensajes a un servidor y recibir respuestas controladas por eventos sin tener que consultar al servidor para una respuesta.

¿Cómo gestiona la memora JavaScript?

Los valores en Javascript se reservan en memoria cunado objetos, string, etc, son creados y automáticamente liberados cunado ya no son utilizados. A este proceso se le llama recolección de basura.
